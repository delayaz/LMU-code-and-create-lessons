Part 11:00 – 1:50

Tweak the robot

You've got a working random walk. Now let's mess with it — small changes to a loop you already understand teach you more than building something new from scratch.

Your turn

Try at least 2 of these changes

  • Weighted coin: make the robot more likely to go right than left. random.choice(["left", "right", "right"]) makes right twice as likely
  • Bigger track: change the track from 21 spaces to 41. What has to change in your code besides just one number? the starting position, the while condition's upper bound, and the track-drawing math all reference the track size
  • Step counter on screen: print the step number next to each line of the track as it draws, e.g. "Step 4: ----------R----------"
  • Two robots: track two positions at once, each with their own independent coin flip, printed on the same line with different letters (R1 and R2).
!If a change makes your program run forever (frozen, no output stopping), that almost always means the while condition can never become false anymore — check that your track-size math and your stopping condition still match after resizing things.
Stretch — done early?

Run 100 walks silently and just report stats

Skip printing every track line. Just run the walk 100 times in a loop, and at the end print the longest walk, the shortest walk, and the average — no track visualization needed for this one, just the numbers.

Part 22:00 – 2:50

Level up: the robot in 2D

Your robot has been stuck on a single line all day. Let's set it loose on a grid — now it can step up, down, left, or right, and it stops once it's far enough from where it started.

main.py
import random

x = 0
y = 0
steps = 0
distance = 0

while distance < 5:
    print(f"Step {steps}: position ({x}, {y})")

    direction = random.choice(["up", "down", "left", "right"])
    if direction == "up":
        y = y + 1
    elif direction == "down":
        y = y - 1
    elif direction == "left":
        x = x - 1
    else:
        x = x + 1

    distance = abs(x) + abs(y)
    steps = steps + 1

print(f"Got 5+ blocks away after {steps} steps! Final spot: ({x}, {y})")
$ Step 0: position (0, 0) Step 1: position (0, 1) Step 2: position (-1, 1) ... (continues until it's far enough away) Got 5+ blocks away after 9 steps! Final spot: (-2, 4)

abs(x) + abs(y) measures how many total blocks away from the start the robot is (this is called "taxicab distance" — like counting city blocks instead of a straight line). The loop keeps going until that distance hits 5 or more.

Your turn

Build the 2D robot

Copy the structure above. Run it a few times and watch how unpredictable the path is — sometimes it wanders close to home for a while before escaping.

  • What happens if you lower the distance target from 5 to 2? it finishes in fewer steps on average, since it doesn't have to wander as far
!Common snag: the variable name distance needs to be recalculated inside the loop, after the position updates — if you calculate it before moving, the while condition checks stale, one-step-old information.
Stretch — done early?

Draw a tiny map

Keep a list of every (x, y) position the robot has visited. After the loop ends, print the full list — can you tell from the list alone whether the robot ever crossed its own path?

Stretch — race two robots

First one to escape wins

Track two robots at once, each with their own x, y, and distance, each taking a random step every round. Use one while loop that keeps going until either robot reaches distance 5, and print which one escaped first (and how many steps each one had taken when it happened).

This needs a condition like while distance1 < 5 and distance2 < 5: — both robots have to still be "trapped" for the loop to keep going. The moment either one escapes, the loop stops, same idea as your single robot, just watching two positions instead of one.
