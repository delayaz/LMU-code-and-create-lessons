Optional · For students who specifically want a GUI

Install Python on your own laptop

Replit is having trouble running GUI libraries (Tkinter) right now. If you specifically want your project to have an actual window with clickable buttons instead of console text, this guide gets Python and a code editor set up on your own laptop, where Tkinter works fine.

✓ This is completely optional. Everyone else keeps building in Replit exactly as before — console-based projects work great there, and nothing is lost by staying on Replit.
Step 1

Install Python

Pick the instructions for your computer.

Windows
  1. Go to python.org/downloads
  2. Click the big "Download Python" button
  3. Run the installer once downloaded
  4. Check "Add Python to PATH" — see callout below
  5. Click "Install Now"
Mac
  1. Go to python.org/downloads
  2. Download the macOS installer
  3. Run it and click through (defaults are fine)
  4. No PATH checkbox needed on Mac
Windows users — don't skip this: on the very first installer screen, there's a checkbox at the bottom that says "Add Python to PATH." Check it before clicking Install. This is the single most common reason Python installs "fail" — skipping it doesn't cause an error during install, but nothing will work afterward.
→ python.org/downloads
Step 2

Confirm it worked

Open a terminal — on Windows, search "Command Prompt" in the Start menu; on Mac, search "Terminal" in Spotlight (Cmd+Space). Type this and press Enter:

terminal
python --version
$ Python 3.12.4

On Mac, if that doesn't work, try python3 --version instead.

!If you see an error instead of a version number, the install didn't fully work — raise your hand rather than troubleshooting alone. This is almost always the "Add to PATH" checkbox on Windows.
Step 3

Install a code editor — VS Code

Python needs somewhere to write code. VS Code is free and works on both Windows and Mac.

  1. Go to code.visualstudio.com
  2. Download for your operating system
  3. Install it, then open it once
→ code.visualstudio.com
Step 4

Test that Tkinter actually works

In VS Code, create a new file named test.py and paste this in:

test.py
import tkinter as tk

root = tk.Tk()
root.title("It works!")
label = tk.Label(root, text="Tkinter is working!", font=("Arial", 20))
label.pack(pady=40, padx=40)
root.mainloop()

Run it — in VS Code, click the ▶ Run button top-right, or right-click in the file and choose "Run Python File in Terminal." A small window should pop up on your screen.

!If the window pops up, you're done — go back to the GUI starter template from class and start building for real. If it doesn't, don't spend more than 10 minutes troubleshooting alone.
● setup guide complete

If this didn't go smoothly

That's completely fine — switch back to Replit and build your project as a console app instead. No project is worse for being console-based; every track works great that way.
