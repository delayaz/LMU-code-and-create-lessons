Burst 19:00 – 10:15

Standup, then protected build time

Start with a 5-minute standup — go around in a circle, each person says what they're building/fixing today and one thing they're stuck on. Then it's real, protected build time. No new lecture today — before you dive in, know what you're aiming for.

Goal — what does "done" look like by tomorrow?

Three things to hit

  1. Core feature works. The main thing your app does actually runs, start to finish, without crashing.
  2. One polish item. Better variable names, a cleaner print statement, an extra input check — something that shows care.
  3. One thing you can talk about. A decision you made, a bug you fixed, or a feature you're proud of — this becomes part of your demo.
Stretch — done early?

Push further

  • Add error handling — what happens if the user types something weird?
  • Add one small second feature that builds on what you already have
  • Go back through your code and clean up variable names so someone else could read it
  • Add comments explaining the trickiest part of your code

Burst 210:15 – 10:45

Design Sprint: Tech for Good

Thirty minutes, different brain. You'll get a real person and a real problem — no code, just ideas — with enough time to actually think it through instead of rushing to a pitch.

Your persona

"Maria, 68, just moved to a new city and doesn't know anyone."

The constraint

You can only use tools that exist right now, or things a beginner coder could realistically build.

How the 30 minutes breaks down
  1. 5 min — Read the persona and constraint aloud as a group. Quick gut-check: what's everyone's first instinct for what Maria needs?
  2. 8 min — In pairs or small groups: get specific about Maria's actual week. Not "loneliness" in the abstract — where does she go, who does she talk to, what's the hardest part of a normal Tuesday for her?
  3. 12 min — Sketch a product or feature that helps. Doesn't need to be technical — a napkin sketch of the idea is enough. Push for one specific detail: what's the very first screen or interaction Maria sees?
  4. 5 min — Each group pitches in under a minute: what it is, who it's for, why it helps Maria specifically. One follow-up question from the room per pitch.
Break
