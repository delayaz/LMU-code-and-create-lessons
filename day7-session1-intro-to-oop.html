A class is a blueprint. An object is the real thing built from it.

You've used dictionaries to bundle related data together — a guest's name and age, a movie's title and rating. A class does something similar, but it's a template you can stamp out multiple times, and later, it can hold actions too, not just data.

main.py
class Character:
    def __init__(self, name, health, attack_power):
        self.name = name
        self.health = health
        self.attack_power = attack_power

hero = Character("Aria", 100, 15)
print(hero.name)
print(hero.health)

villain = Character("Grub", 80, 10)
print(villain.name)
print(villain.health)
$ Aria 100 Grub 80

Walk through this slowly: class Character: defines the blueprint. __init__ is a special function that runs automatically every time you create a new Character — it's where you set up that character's own personal values. self means "this specific character" — it's how Python keeps Aria's health separate from Grub's health, even though they're both Characters.

Your turn — Step 1

Build your own Character class

Copy the class above. Create at least 3 different characters with different names, health, and attack power. Print each one's name and health to confirm they're genuinely separate.

  • What happens if you create a 4th character but forget to give it an attack_power? Python raises a TypeError — __init__ requires every parameter you defined unless you give it a default value
!Common snag: forgetting self. before a variable name inside the class — without it, the value doesn't actually attach to the character, it just disappears when the function ends.

Methods — actions a class can do

Attributes are the data a character has. Methods are things a character can do. A method is just a function written inside the class, and it can use and change that character's own attributes.

main.py
class Character:
    def __init__(self, name, health, attack_power):
        self.name = name
        self.health = health
        self.attack_power = attack_power

    def attack(self, target):
        target.health = target.health - self.attack_power
        print(f"{self.name} attacks {target.name} for {self.attack_power} damage!")

    def is_alive(self):
        return self.health > 0

    def status(self):
        print(f"{self.name}: {self.health} HP")


hero = Character("Aria", 100, 15)
villain = Character("Grub", 80, 10)

hero.status()
hero.attack(villain)
villain.status()
$ Aria: 100 HP Aria attacks Grub for 15 damage! Grub: 65 HP

Notice hero.attack(villain) — Aria's attack method takes another character (target) as an argument, and changes that character's health, not her own. This is the real payoff of bundling things into a class: hero and villain can interact with each other using their own built-in behavior.

Your turn — Step 2

!Common snag: writing a method but forgetting to put self as its first parameter — every method inside a class needs self first, even if it doesn't seem to "use" it directly elsewhere in that method.

Put it together — a real battle loop

You already know while loops from Week 1. Combine that with what you just built, and two characters can fight until one of them runs out of health — a real, complete mini-program.

main.py
hero = Character("Aria", 100, 15)
villain = Character("Grub", 80, 12)

round_number = 1
while hero.is_alive() and villain.is_alive():
    print(f"\n--- Round {round_number} ---")
    hero.attack(villain)
    if villain.is_alive():
        villain.attack(hero)
    hero.status()
    villain.status()
    round_number += 1

if hero.is_alive():
    print(f"\n{hero.name} wins!")
else:
    print(f"\n{villain.name} wins!")
$ --- Round 1 --- Aria attacks Grub for 15 damage! Grub attacks Aria for 12 damage! Aria: 88 HP Grub: 65 HP --- Round 2 --- Aria attacks Grub for 15 damage! Grub attacks Aria for 12 damage! Aria: 76 HP Grub: 50 HP ... (continues until someone's health hits 0) Aria wins!
