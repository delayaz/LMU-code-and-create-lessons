Part 11:00 – 1:50

Your first branching story

A choose-your-own-adventure is just print() for the story, input() for the choice, and if/elif/else to decide what happens next.

main.py
print("You wake up in a strange forest. There's a path to the left and a cave to the right.")
choice = input("Go left or right? ")

if choice == "left":
    print("You follow the path and find a small village.")
elif choice == "right":
    print("You step into the cave. It's pitch black.")
else:
    print("You stand still, unsure what 'left' or 'right' even means here.")

Write your own opening scene + first choice

Any setting you want — a spaceship, a haunted house, a locker room before the big game. One print() to set the scene, one input() for the choice, and at least 2 elif branches with different outcomes.

  • How do you handle someone typing "Left" with a capital L if your code checks lowercase "left"? compare using choice.lower() == "left" so capitalization doesn't matter
!Common snag: typing "Left" with a capital L when your code checks for lowercase "left" — Python treats these as different strings. One fix: compare using choice.lower() == "left" so it doesn't matter how they typed it.

Part 22:00 – 2:50

Keep branching

Once the first choice works, add a second decision point after one of the branches — this is exactly how the "interactive story" project track in Week 2 grows, one choice at a time.

Your turn

Add a second choice after one branch

Pick one of your outcomes from Part 1 and add a follow-up question with its own if/elif/else. By the end, your story should have at least 2 decision points and 4+ possible endings.

  • Stuck on structure? Sketch it on paper first — boxes for each scene, arrows for each choice — before typing anything.
Stretch — nest it properly

Put the second choice inside the first branch

Instead of writing the second question as its own separate if/elif block after the first one ends, nest it directly inside the branch it belongs to — an if living inside the elif for "right," for example. This mirrors how the morning's VIP bouncer worked, and it's structurally closer to how branching dialogue trees are built in real games.
