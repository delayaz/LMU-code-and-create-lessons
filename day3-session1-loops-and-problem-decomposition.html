for loops — when you know how many times

A for loop repeats code a set number of times. Yesterday your bouncer checked one guest at a time. Today: ask how many guests are in the group, then loop that many times, running each guest through the same bouncer check.

main.py
group_size = int(input("How many guests are with you? "))

for guest_number in range(group_size):
    print(f"--- Guest {guest_number + 1} ---")
    age = int(input("  Age? "))
    has_ticket = input("  Has a ticket? (yes/no) ") == "yes"

    if age >= 18 and has_ticket:
        print("  -> Come on in.")
    elif age >= 13 and not has_ticket:
        print("  -> You need a ticket first.")
    else:
        print("  -> Sorry, you can't come in.")
$ How many guests are with you? 2 --- Guest 1 --- Age? 17 Has a ticket? (yes/no) no -> You need a ticket first. --- Guest 2 --- Age? 22 Has a ticket? (yes/no) yes -> Come on in.

Notice — everything inside the loop (asking age, asking about the ticket, checking the conditions) is the exact same bouncer logic from yesterday. You didn't rewrite it for each guest — range(group_size) just runs that block however many times the group needs.

while loops — when you don't know how many times

A while loop keeps running as long as a condition is true — you don't know in advance how many times it'll repeat. Let's build a robot on a number line that takes a random step left or right, and keeps walking until it falls off either edge.

main.py
import random

position = 10  # middle of a 21-space track (0 to 20)
steps = 0

while position >= 0 and position <= 20:
    track = "-" * position + "R" + "-" * (20 - position)
    print(track)

    flip = random.choice(["left", "right"])
    if flip == "left":
        position = position - 1
    else:
        position = position + 1

    steps = steps + 1

print(f"Fell off the track after {steps} steps!")
$ ----------R---------- -----------R--------- ----------R---------- ---------R----------- ... (continues until it falls off either end) Fell off the track after 14 steps!

Walk through this line by line with us before typing: random.choice([...]) picks one item from a list at random. The while condition checks the robot's position every time before running the loop body again — the moment position goes below 0 or above 20, the loop stops. Nobody — not even the computer — knows in advance how many steps it'll take.

!Common snag: forgetting to update position inside the loop — if the robot's position never changes, the while condition never becomes false, and the loop runs forever. If your program seems frozen, that's almost always what happened — stop it and check that position actually changes every time through.
