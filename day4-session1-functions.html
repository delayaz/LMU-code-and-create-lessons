Turn the bouncer check into a function

A function is a named, reusable block of code. You give it a name, tell it what information it needs (its parameters), and it can hand back an answer (its return value). Let's package yesterday's bouncer check into one.

main.py
def check_guest(age, has_ticket):
    if age >= 18 and has_ticket:
        return "Come on in."
    elif age >= 13 and not has_ticket:
        return "You need a ticket first."
    else:
        return "Sorry, you can't come in."

# calling the function — this is where it actually runs
result = check_guest(17, False)
print(result)

print(check_guest(22, True))
$ You need a ticket first. Come on in.

Notice — the code between def check_guest(...): and the end doesn't run until you actually call it (like check_guest(17, False)). Defining a function just teaches Python the recipe; calling it is what actually runs it. And return hands back an answer you can store or print — it's different from print(), which just displays something and doesn't give you anything back to use later.

Turn the robot walk into a function

Yesterday, running the robot walk twice meant copying the whole block of code. With a function, you write it once and call it as many times as you want — which is exactly what yesterday's "loop the loop" stretch was secretly setting you up for.

main.py
import random

def random_walk():
    position = 10
    steps = 0
    while position >= 0 and position <= 20:
        flip = random.choice(["left", "right"])
        if flip == "left":
            position = position - 1
        else:
            position = position + 1
        steps = steps + 1
    return steps

# now calling it 5 times is one line, not five copies of the walk
for walk_number in range(5):
    result = random_walk()
    print(f"Walk {walk_number + 1}: {result} steps")
$ Walk 1: 14 steps Walk 2: 6 steps Walk 3: 23 steps Walk 4: 9 steps Walk 5: 17 steps

This is the same "loop calling a function" shape you'll use constantly — a for loop handles "how many times," and the function handles "what happens each time." Each one only has to think about its own job.
