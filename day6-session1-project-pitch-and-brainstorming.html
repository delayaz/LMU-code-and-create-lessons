Part 29:30 – 9:55

Quick brainstorm — get to one idea fast

Two fast steps to land on a specific idea. We're keeping this short on purpose — the real learning today happens once you're building, not while you're still planning.

Step 1 — 10 min

Write down 5 ideas in your track, don't judge them yet

  • Game track: 5 different specific games — not just "a game"
  • Quiz track: 5 different quiz topics — what would you actually want to take?
  • Story track: 5 different settings or situations — a haunted house, a locker room, a spaceship...
  • Data track: 5 different things you could collect data about — your own habits, sports stats, song lengths...
Step 2 — 10 min

Narrow to one — two filters

  • Can I picture the whole thing? If you can't imagine someone using it start to finish, set it aside.
  • Do I actually want to spend 3 days on this? Not "is it impressive" — just, would this hold your attention.

Whatever survives both, circle it. That's your project — you'll build the actual details as you go, not all on paper first.

!If you're stuck between two ideas, just pick one. A specific, finished small project beats a vague, ambitious plan every time in a course this short.

New skill — Step C

Make it run more than once — the replay loop

Right now your program runs once and stops. Almost every real app — games, quizzes, tools — lets you do something again without restarting the whole program. That's just a while loop wrapped around what you already have, plus one question at the end.

main.py — the replay loop pattern
playing = True

while playing:
    # --- everything you already built goes here, indented one level in ---
    print("...your existing code runs here...")

    # --- then, ask if they want to go again ---
    again = input("\nGo again? (yes/no) ")
    if again.lower() != "yes":
        playing = False

print("Thanks for playing!")
Your turn

Wrap your skeleton in a replay loop

Take the code you just built for your track and indent it one level inward, inside a while playing: loop like the pattern above. Add the "go again?" question at the end.

  • Game track: this means a whole new round, with a new random number, every time they say yes.
  • Quiz App track: this could replay the same questions, or reset the score back to 0 for a fresh attempt.
  • Interactive Story track: this lets someone try a different choice and see the other path, without restarting the program.
  • Data Tool track: this could let them analyze a second set of numbers right after the first, without re-running the file.
!Common snag: forgetting to indent your existing code to match the new while loop — every line that should repeat needs to line up one level deeper than while playing: itself.

Stretch — done early?

Add a counter to your replay loop

Track how many times they've played/answered/checked in a variable that goes up each time through the loop, and print it in your closing message — "Thanks for playing! You played 3 rounds." Small addition, but it makes the loop feel like it's actually keeping track of something.
