Lists — many values, kept in order

A list holds multiple values in a single variable, in order, indexed starting from 0. You’ve already looped over a range() of numbers — now you’ll loop over a list of real things.

main.py
guest_list = []

guest_list.append("Jesus")
guest_list.append("Kellan")
guest_list.append("Cassie")

print(guest_list)
print(guest_list[0])
print(len(guest_list))

for name in guest_list:
    print(f"Checking in: {name}")
$ ['Jesus', 'Kellan', 'Cassie']
Jesus
3
Checking in: Jesus
Checking in: Kellan
Checking in: Cassie

Three new things at once: [] creates an empty list, .append(...) adds something to the end, and list[0] grabs the item at position 0 (lists start counting from 0, not 1). The for name in guest_list: loop is the same shape as your Day 3 for loops — it just hands you real items now instead of numbers.

Dictionaries — values you look up by name

A list finds things by position (0, 1, 2…). A dictionary finds things by a label you choose — a key — paired with a value. Perfect for storing more than just a name per guest.

main.py
guest = {
    "name": "Elin",
    "age": 16,
    "has_ticket": True
}

print(guest["name"])
print(guest["age"])

# add a new key, just by assigning to it
guest["vip"] = False
print(guest)
$ Elin
16
{'name': 'Elin', 'age': 16, 'has_ticket': True, 'vip': False}

Now combine both: a list of dictionaries is how you’d really store a whole guest list with full details on each person — each item in the list is one guest’s dictionary.

main.py
guests = [
    {"name": "Elin", "age": 16, "has_ticket": True},
    {"name": "Yutaro", "age": 17, "has_ticket": False}
]

for guest in guests:
    if guest["age"] >= 18 and guest["has_ticket"]:
        print(f"{guest['name']}: come on in")
    else:
        print(f"{guest['name']}: not yet")
$ Elin: not yet
Yutaro: not yet

This is the cleanest version yet of your bouncer — yesterday it was a function taking separate age and has_ticket arguments. Now one dictionary holds everything about a guest, and one list holds everyone.

Now wrap an operation in a function. This filter_by_genre function takes the whole playlist and a genre string, and returns a new list containing only the matching songs — the original playlist is untouched.

main.py — filter function
def filter_by_genre(playlist, genre):
    result = []
    for song in playlist:
        if song["genre"] == genre:
            result.append(song)
    return result

pop_songs = filter_by_genre(playlist, "pop")
for song in pop_songs:
    print(song["title"])

RPG character sheet — a dictionary that does things

A dictionary doesn’t have to just store data — you can write functions that take a dictionary as a parameter and modify it. This is how real programs pass structured data around. Your character is a dictionary; your functions are the things that can happen to them.

main.py — the character
hero = {
    "name": "Zara",
    "hp": 100,
    "max_hp": 100,
    "attack": 15,
    "defense": 8,
    "inventory": ["health potion", "iron sword"]
}

Here’s one function already written for you. Notice: it takes the whole dictionary as a parameter — no separate hp, defense arguments — and it modifies the dictionary directly. After calling it, hero["hp"] has actually changed.

main.py — pre-written: take_damage
def take_damage(character, amount):
    # defense reduces the hit — can't go below 0
    actual_damage = max(0, amount - character["defense"])
    character["hp"] -= actual_damage
    print(f"{character['name']} takes {actual_damage} damage!  HP: {character['hp']}/{character['max_hp']}")

take_damage(hero, 20)
$ Zara takes 12 damage! HP: 88/100

Load a CSV — your list of dictionaries, at scale

A CSV (comma-separated values) file is one of the most common ways data lives in the real world — spreadsheets, exports, datasets. Python’s built-in csv module can read one and hand you back exactly the structure you built today: a list of dictionaries, one per row, with the column headers as keys.

  3. Name it exactly games.csv — the .csv extension is important, not .py.
  4. Hit Copy data below, then paste everything into that new file.

main.py
import csv
 
games = []
 
with open("games.csv") as file:
    reader = csv.DictReader(file)
    for row in reader:
        games.append(row)
 
# games is now a list of dictionaries — same structure as today
print(games[0])
print(games[0]["title"])
print(len(games), "games loaded")

DictReader reads the first row as key names and every row after that as a dictionary — no manual work. The with open(...) as file: block handles opening and closing the file automatically. One thing to watch: every value comes in as a string, even numbers. See the "9.5" and "2022" in the output above — they’re text until you convert them.

main.py — converting types
