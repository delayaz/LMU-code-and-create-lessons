if, elif, else — your code picks a path

An if statement runs a block of code only when something is true. Add elif for more options, and else to catch everything left over.

main.py
score = 85

if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
else:
    grade = "F"

print(f"A score of {score} is a {grade}.")
$ A score of 85 is a B.

Notice — Python checks each condition top to bottom, and stops at the first one that's true. That's why order matters here.

Your turn

Build a grade calculator

Use input() to ask for a score, convert it to a number, then use if/elif/else to print the letter grade.

  • Remember: input() always returns text — wrap it in int(...) to turn it into a real number you can compare with >=.
  • Done fast? Add a +/- system — 87 is a "B+", 81 is a "B-".
!Common snag: forgetting int() around the input, so Python tries to compare text to a number and throws a TypeError. Also watch indentation — every line inside an if block needs the exact same indent.

and, or, not — combine conditions

Sometimes one condition isn't enough. and needs both sides true. or needs just one. not flips true to false.

main.py
age = 17
has_ticket = True

if age >= 18 and has_ticket:
    print("Come on in.")
elif age >= 13 and not has_ticket:
    print("You need a ticket first.")
else:
    print("Sorry, you can't come in.")
$ You need a ticket first.

!Common snag: mixing up when you need and vs or — read it out loud as a sentence first ("let them in if they're 18 and have a ticket") before writing the code.
● AI Segment #1

How do tools like ChatGPT and Claude actually work?

You're going to be tempted to use AI tools the rest of this course — so let's set the norms now, before any problem comes up. A quick mental model: these tools predict the next most likely word, over and over, based on patterns in huge amounts of text. They're not "thinking" the way you are, and they're not always right.

Good use

Asking AI to explain an error message in plain language, or to suggest a few different directions for a project idea you're stuck on.

Lazy use

Asking AI to just write the whole thing for you — you'll have code that runs, but you won't understand it, and you can't fix it when it breaks.

Rule of thumb for this course: AI is a tutor, not a typist. If you can't explain what a line of code does after asking AI for help with it, slow down and ask a human instead — Delaram, Lauren, or a neighbor.
